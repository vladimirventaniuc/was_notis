Usability Test Plan Section

Test Plan for: Visia Chrome Extension

Persona Type Tested: Served

Context: a quiet room, an environment where the user is comfortable without distractions
Duration: 15 minutes

Complete the following before starting: Greeting and Consent & User Background Information.

Overall Objectives

The objectives of the usability test are to check how easy is for a person to interact with the extension. In this test we will test:

  • how intuitive is the interface
  • how often the user encounters an obstacle when trying to accomplish a task
  • the time spent doing all the tasks
  • how useful is the extension in user perception

Finally, we will collect user responses and we will try to improve user experience and to add new features that users consider relevant for an extension of this type.

Scenario

User receives a laptop with the Chrome browser installed and the visia extension added in Chrome. He has to navigate to a page and to learn how to use the extension.

Tasks
  • Task 1: Open any webpage on browser (1 min)
  • Task 2: Left click on extension icon an observ all the buttons (2 min)
  • Task 3: Select a portion of text and click on play button (2 min)
  • Task 4: Use pause and stop buttons to stop the reading of text (2 min)
  • Task 5: Change a color for text and background and observ how page content is modified (2 min)
  • Task 6: Navigate to option page and change any option for text reading. Retry the action from task 3 and observ the change (3,5 min)
  • Task 7: From options page check the box for paragraph for both background and text (make sure all other checkboxes are unchecked). Retry the action from task 5 and observ the change (3,5 min)
Post-test Questions
  • How difficult was to find and access the components of the extension?
  • Was difficult to associate button icons with the desired actions?
  • What would you change about the extension?
Notes for Test Coordinator
  • Remind the user that you would like them to 'think aloud' so that you can understand what they are thinking, about the task or the product/application;
  • Read the user the scenario for the test;
  • Read the user the tasks he has to complete. If the tasks are complex, you may want to put a sheet of paper in front of the user with each task written out;
  • Record the user's actions, comments, questions, and body language (write them in the Test Observations section accompanied by video/audio).
  • Don't offer help; let the user attempt to perform the task. If they ask for help reply with: "What do you think you/that would do?" "What do you think that means?".
